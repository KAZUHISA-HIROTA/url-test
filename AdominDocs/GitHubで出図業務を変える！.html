# 映里奈さんの作業（修正 → コミット → プッシュ） 【1. 修正】 VS Code で auto_shutsuzu.py を開く Claude の修正コードをテキストで貼り付け・手直し ファイルを保存（Ctrl+S） ↓ 【2. コミット（セーブ）📝】 VS Code 左「ソース管理」を開く → 変更ファイル auto_shutsuzu.py が表示される → メッセージ欄に入力： 「図番桁数を10桁対応に修正（岡部くん要望）」 → 「コミット」ボタンをクリック ↓ 【3. プッシュ（送信）⬆️】 「同期（プッシュ）」ボタンをクリック ↓ ✅ GitHub の引き出しに最新版が入った！ ✅ 岡部くんと光博さんに届く準備完了！

💡 コミットメッセージのコツ（スギヤス版）

  • 「誰の要望で変えたか」を書くと後で責任の所在がわかる（例：岡部くん要望、光博さん要望）
  • 「AI生成」「廣田確認済」などのタグをつけると管理しやすい
  • これで 「どのバージョンがどの改造案件に対応しているか」 が一目でわかる

🔔 プッシュ・プルのポイント（複数人利用）

  • 映里奈さんがプッシュした瞬間から、岡部くん・光博さんは最新版をプルできる状態になる
  • プルの操作は岡部くんと光博さんで独立して行える。どちらかが先にプルしても問題ない
  • 岡部くんがプルしていない間に光博さんがプルしても、どちらも同じ最新版が手に入る
  • 廣田マネージャーはプルしなくてもGitHub画面から最新の変更内容を確認できる

# バグ発生 → 復旧の手順 【1. 履歴確認】 GitHub のリポジトリページ → 「Commits」をクリック → 全コミット履歴が時系列で表示される → 壊れたタイミングの修正が一目でわかる 【2. 変更差分の確認】 問題のコミットをクリック → 何行目を何から何に変えたかが 　 赤（削除）・緑（追加）で表示される 【3. 一個前のバージョンに戻す】 「Revert（元に戻す）」を選択 → 映里奈さんがプッシュ → 岡部くん・光博さんがプル ✅ 5分で正常なバージョンに復旧完了！
